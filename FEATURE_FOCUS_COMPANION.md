## 🎯 功能目标

用户设定任务后，AI 定期观察屏幕，判断是否在专心工作。走神时温和提醒，结束时生成专注/走神简报。

---

## 🏁 关键里程碑

### 里程碑 A（PR #6 合并后）

**可演示场景**: `开始专注 → AI 看到走神 → 发出提醒`

```
用户操作:
  1. 开启屏幕共享
  2. 点击「开始专注」→ 输入"写数据库模块" / 30 分钟
  3. 在 VS Code 中正常写代码 → AI 静默
  4. 切换到 B 站看视频 → 约 2 分钟后 AI 发出提醒:
     "看起来你切到了 B 站，还在写数据库模块吗？"
  5. 用户切回 VS Code → AI 简短肯定

包含的 PR:
  ✅ PR #1  后端：专注模式数据模型 + API 骨架（mock）
  ✅ PR #2  前端：感知哈希工具函数
  ✅ PR #3  前端：专注模式 UI 入口 + 设置面板
  ✅ PR #4  后端：走神检测状态机
  ✅ PR #5  后端：AI 判断 + 提醒文案
  ✅ PR #6  前端：轮询循环 + 提醒展示
```

### 里程碑 B（PR #8 合并后）

**可演示场景**: `开始专注 → 被提醒 → 结束 → 看简报卡片`

```
用户操作:
  1-4. 同上
  5. 结束 30 分钟专注（用户点击「结束专注」或时间到）
  6. 聊天面板出现简报卡片:
     ┌──────────────────────────────┐
     │ 📊 专注 22 分钟              │
     │ 📱 走神 6 分钟（B站、知乎）    │
     │ 🚶 离开 2 分钟               │
     │                              │
     │ 📋 时间线（彩色标记）          │
     │ 📝 AI 总结 + 💡 小建议        │
     └──────────────────────────────┘

包含的 PR:
  ✅ PR #1~#6（里程碑 A 的全部）
  ✅ PR #7  后端：任务简报生成 API
  ✅ PR #8  前端：简报卡片展示 + 结束体验
```
